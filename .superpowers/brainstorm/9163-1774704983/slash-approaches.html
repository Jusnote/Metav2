A) Inline Autocomplete

Sem popover. Dropdown aparece direto embaixo do input, como Google Search. Um input so. /banca vira chip colorido inline. Resultados filtram em tempo real. Hints de teclado no rodape.

Pros

  • Zero confusao — um input, um dropdown
  • Familiar (autocomplete)
  • Sem duplicacao

Cons

  • Nao reutiliza o popover do pill click
  • Sem progress bars / checkboxes

B) Popover Limpo (sem search input)

Mesmo popover do pill click, mas sem o campo de busca. Banner amarelo indica "/ banca — digitando no input principal". Mantem progress bars e checkboxes. Input principal controla tudo.

Pros

  • Reutiliza o popover existente
  • Com progress bars, checkboxes, recentes
  • Um input so

Cons

  • Popover longe do input (visualmente)

C) Command Palette

O input se transforma quando digita /. Chip / banca aparece no inicio, cursor fica apos ele. Resultados aparecem dentro do mesmo container, abaixo do input. Um unico bloco visual coeso. Estilo VS Code / Notion / Linear.

Pros

  • Mais clean e coeso possivel
  • Input + resultados no mesmo container
  • Familiar pra devs (VS Code, Notion)
  • Zero duplicacao

Cons

  • Search bar muda de forma (pode confundir no inicio)
  • Precisa de Esc pra sair
