CREATE EXTENSION IF NOT EXISTS unaccent;
DO $$ BEGIN
  CREATE TEXT SEARCH CONFIGURATION pt_unaccent (COPY = portuguese);

## Implementation Notes

### Parent-child for revoked items
Revoked dispositivos get parent-child relationships just like active ones. The structural position is preserved regardless of revocation status.

### Laws without ARTIGO structure
Some laws (decrees, portarias) have no ARTIGO items. In these cases, `currentArtigo` stays null, and all items get `parent_id = null`, `artigo_id = null`. This is correct behavior.

### API changes (deferred)
The GraphQL API does not yet expose `parent_id`, `artigo_id`, `depth`. These fields exist in the DB and can be exposed later by adding them to the schema and resolver. Not in scope for this plan.

### 1M+ bulk processing (separate cycle)
After validating with CP + CC:
1. DROP GIN indexes
2. Run `process.js --all --force` across D:/leis
3. Run `upload.js --all --pg $DATABASE_URL`
4. CREATE GIN indexes
This is a multi-hour operation — run on the Hetzner server directly for speed.
